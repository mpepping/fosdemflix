## fosdemflix

A Netflix-like browser for FOSDEM conference videos (2020–2026). Built as a fully static site — no server required.

## Tech Stack

- **Frontend**: Vite + TypeScript (vanilla, no framework)
- **Search**: MiniSearch (client-side)
- **Data**: Build-time fetch of FOSDEM schedule XMLs → `public/data/talks.json`

## Project Structure

```
fosdemflix/
├── scripts/
│   └── fetch-data.ts       # Fetches FOSDEM XMLs → public/data/talks.json
├── src/
│   ├── main.ts             # App entry: rendering, filter/search wiring
│   ├── types.ts            # Talk, FilterState interfaces
│   ├── search.ts           # MiniSearch index setup and query logic
│   ├── player.ts           # HTML5 video modal
│   ├── filters.ts          # Year/track filter logic
│   └── styles/
│       ├── main.css        # Base styles, CSS variables, layout
│       ├── cards.css       # Video card grid
│       ├── player.css      # Video player modal
│       └── search.css      # Search bar and filter pills
├── public/
│   └── data/
│       └── talks.json      # Generated by fetch-data (committed)
└── index.html
```

## Commands

| Command | Description |
|---------|-------------|
| `npm run fetch-data` | Refresh `public/data/talks.json` from FOSDEM schedule XMLs |
| `npm run dev` | Start local dev server |
| `npm run build` | Build static site to `dist/` |
| `npm run preview` | Preview the production build |

## Data Sources

| Year | URL |
|------|-----|
| 2020–2025 | `https://archive.fosdem.org/{year}/schedule/xml` |
| 2026 | `https://fosdem.org/2026/schedule/xml` |

FOSDEM XML `<link>` elements use `href` attributes without `type` — video files are identified by URL extension on `video.fosdem.org`:
- `.mp4` → `videoMp4`
- `.webm` → `videoWebm`
- `.vtt` → `subtitles`

Only talks with at least one video URL are included. Current dataset: ~5,600 talks.

## Key Conventions

- Talks display newest-first (2026 → 2020)
- Search boosts: title ×3, speakers ×2, track ×1.5, abstract ×1
- Year + track filters combine with AND logic; search is independent
- `public/data/talks.json` is committed so the app works without re-fetching
